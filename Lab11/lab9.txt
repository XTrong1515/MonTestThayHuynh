9.3 BasePage - Lớp nền tảng cho m ọi Page Object  
BasePage đóng gói t ất cả các thao tác WebDriver ph ổ biến thành các method có Explicit 
Wait tích h ợp sẵn. M ọi Page Object đ ều extends BasePage đ ể dùng l ại các method này : 
// BasePage.java  
package framework.base;  
 
public abstract class BasePage {  
    protected WebDriver driver;  
    protected WebDriverWait wait;  
 
    public BasePage(WebDriver driver) {  
        this.driver = driver;  
        // Explicit wait 15 giây — đủ cho mạng chậm, không ch ờ lãng phí  
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(15));  
        PageFactory.initElements(driver, this);  // Kh ởi tạo @FindBy annotations  
    } 
 
    // Chờ element clickable r ồi mới click — tránh ElementNotInteractableException  
    protected void waitAndClick(WebElement element) {  
        wait.until(ExpectedConditions.elementToBeClickable(element)).click();  
    } 
 
    // Xóa n ội dung cũ r ồi gõ — tránh dữ liệu bị nối thêm  
    protected void waitAndType(WebElement element, String text) {  
        wait.until(ExpectedConditions.visibilityOf(element));  
        element.clear();  
        element.sendKeys(text);  
    } 
 
    // Lấy text đã trim whitespace  
    protected String getText(WebElement element) {  
        return wait.until(ExpectedConditions.visibilityOf(element)).getText().trim();  
    }

 9.5 BaseTest - Quản lý vòng đ ời WebDriver  
BaseTest đ ảm nh ận việc tạo và d ọn dẹp WebDriver cho m ỗi test method. Dùng 
ThreadLocal đ ể hỗ trợ chạy song song an toàn (đã h ọc và làm ở Lab trước): 
// BaseTest.java  
package framework.base;  
 
public abstract class BaseTest {  
    private static ThreadLocal<WebDriver> tlDriver = new ThreadLocal<>();  
 
    protected WebDriver getDriver() { return tlDriver.get(); }  
 
    @Parameters({"browser", "env"})  
    @BeforeMethod(alwaysRun = true)  
    public void setUp(@Optional("chrome") String browser,  
                      @Optional("dev") String env) {  
        // Đặt env làm System property đ ể ConfigReader đ ọc đúng file  
        System.setProperty("env", env);  
 
        WebDriver driver = DriverFactory.createDriver(browser);  
        driver.manage().window().maximize();  
        driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(5));  
        driver.get(ConfigReader.getInstance().getBaseUrl());  
        tlDriver.set(driver);  
    } 
 
    @AfterMethod(alwaysRun = true)  
    public void tearDown(ITestResult result) {  
        // Chụp ảnh TRƯỚC khi quit — bắt buộc trong d ự án thực để debug  
        if (result.getStatus() == ITestResult.FAILURE) {  
            String screenshotPath = ScreenshotUtil.capture(getDriver(), 
result.getName());  
            // Đính kèm vào Allure report n ếu đang dùng Allure  
            Allure.addAttachment("Screenshot khi fail",  
                new ByteArrayInputStream(ScreenshotUtil.captureAsBytes(getDriver())));  
        } 
        if (getDriver() != null) {  
            getDriver().quit();  
            tlDriver.remove();  // Quan tr ọng: tránh memory leak khi ch ạy song song  
        } 
    } 
}

❖ Tại sao ph ải tách d ữ liệu test ra kh ỏi code Java?  
Trong d ự án th ực tế, QA Lead qu ản lý hàng trăm test case. Ngư ời viết test data  
thường không ph ải là ngư ời viết code — đó có th ể là QA không bi ết lập trình,  hoặc 
Business Analyst cung c ấp dữ liệu nghi ệp vụ. 
Nếu dữ liệu nằm trong code Java:  
  → M ỗi lần thêm/s ửa test case ph ải build l ại project (m ất 2-5 phút)  
  → Ngư ời không bi ết Java không c ập nhật được 
  → D ữ liệu production (m ật khẩu thật) có th ể lọt vào git repository  
Nếu dữ liệu nằm trong file Excel/JSON:  
  → QA m ở Excel, thêm dòng m ới → ch ạy lại test ngay, không c ần build  
  → D ữ liệu nhạy cảm không commit vào git (thêm vào .gitignore)  
  → D ễ audit: ai thêm/s ửa test case khi nào

            Sheet sheet = workbook.getSheet(sheetName);

    // Xử lý từng kiểu dữ liệu trong cell  
    private static String getCellValue(Cell cell) {  
        if (cell == null) return "";  // Cell r ỗng → chu ỗi rỗng, không 
NullPointerException  
        return switch (cell.getCellType()) {  
            case STRING  -> cell.getStringCellValue().trim();  
            case NUMERIC -> String.valueOf((long) cell.getNumericCellValue());  
            case BOOLEAN -> String.valueOf(cell.getBooleanCellValue());  
            case FORMULA -> cell.getCachedFormulaResultType() == CellType.NUMERIC  
                           ? String.valueOf((long) cell.getNumericCellValue())  
                           : cell.getStringCellValue();  
            default      -> "";  
        }; 
    } 
}

 // RetryAnalyzer.java  
public class RetryAnalyzer implements IRetryAnalyzer {  
    private int retryCount = 0;  
 
    @Override  
    public boolean retry(ITestResult result) {  
        int maxRetry = ConfigReader.getInstance().getRetryCount();  
        if (retryCount < maxRetry) {  
            retryCount++;  
            System.out.println("[Retry] Đang ch ạy lại lần " + retryCount  
                               + " cho test: " + result.getName());  
            return true;   // true = ch ạy lại 
        } 
        return false;      // false = d ừng, đánh d ấu FAIL  
    } 
} 
 
// RetryListener.java — Áp dụng RetryAnalyzer cho m ọi test t ự động 
public class RetryListener implements IAnnotationTransformer {  
    @Override  
    public void transform(ITestAnnotation annotation, Class testClass,  
                          Constructor testConstructor, Method testMethod) {  
        annotation.setRetryAnalyzer(RetryAnalyzer.class);  
        // Không c ần thêm retryAnalyzer = RetryAnalyzer.class vào t ừng @Test  
    } 
}
